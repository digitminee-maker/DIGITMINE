# نظام إدارة العائلات - Family Management System

## نظرة عامة

تم تطوير نظام جديد لعرض المنتجات بناءً على **العائلات** (Families) بدلاً من عرض كل منتج بشكل منفرد. هذا يسمح للعملاء برؤية جميع العروض المتاحة لخدمة معينة (مثل Netflix) في مكان واحد.

## المميزات الرئيسية

### 1. واجهة العميل (Customer Interface)
- **عرض العائلات**: يمكن للعملاء رؤية العائلات (مثل Netflix, Spotify, Adobe) كبطاقات قابلة للتوسع
- **التوسع لرؤية العروض**: عند الضغط على عائلة، يتم عرض جميع المنتجات والعروض المتاحة
- **تبديل الوضع**: يمكن التبديل بين عرض العائلات وعرض المنتجات الفردية
- **البحث والتصفية**: البحث يعمل على مستوى العائلات والمنتجات

### 2. لوحة تحكم المسؤول (Admin Dashboard)
- **صفحة جديدة**: `/admin/families` لإدارة جميع العائلات والعروض
- **عرض منظم**: تجميع المنتجات حسب العائلة مع إمكانية التوسع
- **تعديل العروض**: يمكن تعديل جميع تفاصيل العرض (السعر، المدة، المخزون، إلخ)
- **إدارة المخزون**: عرض وتحديث المخزون لكل عرض
- **التحكم بالرؤية**: إظهار/إخفاء العروض بسهولة

## البنية التقنية

### الملفات الجديدة

#### 1. `src/lib/family-catalog-data.ts`
مكتبة لجلب وتجميع البيانات حسب العائلة:
- `useFamilyCatalog()`: جلب جميع العائلات مع منتجاتها وعروضها
- `useFamily()`: جلب عائلة محددة مع تفاصيلها الكاملة
- `ProductFamily`: نوع البيانات للعائلة

```typescript
type ProductFamily = {
  family: string;
  products: CatalogProduct[];
  minPrice: number | null;
  maxPrice: number | null;
  totalStock: number;
  image: string | null;
};
```

#### 2. `src/components/FamilyCard.tsx`
مكون بطاقة العائلة القابلة للتوسع:
- عرض صورة العائلة والمعلومات الأساسية
- قائمة المنتجات عند التوسع
- معاينة العروض والأسعار
- رابط لعرض جميع العروض

#### 3. `src/routes/admin.families.tsx`
صفحة إدارة العائلات الجديدة:
- عرض جميع العائلات والمنتجات
- تعديل تفاصيل العروض مباشرة
- حذف العروض
- إظهار/إخفاء العروض
- البحث والتصفية

#### 4. `scripts/import-csv-data.ts`
سكريبت لاستيراد بيانات CSV:
- قراءة ملف CSV
- تجميع البيانات حسب العائلة
- إنشاء منتج واحد لكل عائلة
- إنشاء عروض لكل منتج

### التعديلات على الملفات الموجودة

#### 1. `src/routes/shop.tsx`
- إضافة `useFamilyCatalog()` لجلب العائلات
- إضافة وضع عرض جديد: `'families'` و `'products'`
- أزرار التبديل بين الوضعين
- تصفية العائلات حسب البحث

#### 2. `src/components/admin/AdminSidebar.tsx`
- إضافة رابط جديد: `/admin/families`
- أيقونة Layers للعائلات

## كيفية الاستخدام

### للعملاء

1. **الذهاب إلى المتجر**: `/shop`
2. **اختيار وضع العرض**: 
   - "العائلات" - لعرض العائلات كبطاقات قابلة للتوسع
   - "المنتجات" - لعرض المنتجات الفردية (الوضع القديم)
3. **البحث**: استخدام شريط البحث للبحث عن عائلة أو منتج
4. **التوسع**: الضغط على بطاقة العائلة لرؤية المنتجات والعروض
5. **الشراء**: الضغط على منتج للذهاب إلى صفحة التفاصيل

### للمسؤول

1. **الذهاب إلى الإدارة**: `/admin/families`
2. **البحث**: استخدام شريط البحث للبحث عن عائلة
3. **التوسع**: الضغط على عائلة لرؤية المنتجات والعروض
4. **التعديل**: الضغط على أيقونة التعديل لتحرير العرض
5. **الحفظ**: الضغط على "حفظ" لحفظ التغييرات
6. **الحذف**: الضغط على أيقونة الحذف لحذف العرض
7. **الرؤية**: الضغط على أيقونة العين لإظهار/إخفاء العرض

## نموذج البيانات

### العلاقات

```
Family (عائلة)
  ├── Product (منتج)
  │   ├── ProductOffer (عرض)
  │   │   ├── name (اسم العرض)
  │   │   ├── duration (المدة)
  │   │   ├── price_dzd (السعر بالدينار)
  │   │   ├── price_usd (السعر بالدولار)
  │   │   ├── stock (المخزون)
  │   │   ├── supplier (المورد)
  │   │   ├── account_type (نوع الحساب)
  │   │   ├── offer_type (نوع العرض)
  │   │   └── active (مفعل)
```

### الجداول في Supabase

- `products`: المنتجات (واحد لكل عائلة)
  - `family` (عائلة): اسم العائلة
  - `name` (اسم): اسم المنتج
  - `slug` (الرابط): رابط فريد
  - `main_image` (الصورة): صورة العائلة
  - `visible` (مرئي): هل المنتج مرئي

- `product_offers`: العروض
  - `product_id` (معرف المنتج): ربط بالمنتج
  - `name` (اسم): اسم العرض
  - `duration` (المدة): مدة الاشتراك
  - `price_dzd` (السعر DA): السعر بالدينار
  - `price_usd` (السعر USD): السعر بالدولار
  - `stock` (المخزون): عدد الوحدات المتاحة
  - `supplier` (المورد): مصدر العرض
  - `account_type` (نوع الحساب): Private/Shared/Family
  - `offer_type` (نوع العرض): Premium/Pro/Standard/etc
  - `active` (مفعل): هل العرض مفعل
